# Plan: iOS Build & Release for NSAT

## Current State

The Flutter project already has an `ios/` scaffold (Runner.xcodeproj, Podfile, AppDelegate.swift, GoogleService-Info.plist, app icons, launch images). The full Dart codebase (46 files across models, providers, services, screens, widgets, theme, routes) is complete and working for Android + Web. Bundle ID is `in.edu.niu.nsat`, target iOS 13.0+.

## Remaining Steps

### 1. Apple Developer Account & Certificates
- Enroll in Apple Developer Program ($99/year) at developer.apple.com if not already done
- Create an App ID for `in.edu.niu.nsat` in the Apple Developer Portal
- Generate a Distribution Certificate (iOS Distribution) in the Developer Portal
- Create a Production Provisioning Profile linked to the App ID + distribution cert
- Register App Store Connect record for NSAT (name, description, screenshots, privacy policy URL)

### 2. Xcode Project Signing Configuration
- Open `ios/Runner.xcworkspace` in Xcode
- Set `DEVELOPMENT_TEAM` to the Apple Team ID in all build configurations (Debug, Profile, Release)
- Switch `CODE_SIGN_IDENTITY` from `"iPhone Developer"` to `"iPhone Distribution"` for Release builds
- Set `CODE_SIGN_STYLE` to `Manual` for Release (or keep Automatic with a paid team)
- Verify `PRODUCT_BUNDLE_IDENTIFIER` = `in.edu.niu.nsat` (already set)
- Set `SWIFT_VERSION` = 5.0 (already set)
- Bump `IPHONEOS_DEPLOYMENT_TARGET` if needed (currently 13.0 — Firebase requires iOS 13+ minimum, so this is fine)

### 3. Firebase iOS Configuration
- `GoogleService-Info.plist` is already present at `ios/Runner/GoogleService-Info.plist` with correct bundle ID `in.edu.niu.nsat`
- Verify Firebase iOS SDK pods will be pulled via `pod install` (Firebase dependencies in pubspec.yaml are cross-platform)
- Ensure `firebase_messaging` (FCM) capabilities are enabled in Xcode: Push Notifications + Background Modes > Remote Notifications
- Add APNs key or certificate in Firebase Console > Project Settings > Cloud Messaging > iOS app configuration

### 4. iOS-Specific Capabilities & Permissions
- In `Info.plist`, add any required usage descriptions (e.g., network access is implicit, but document that no camera/mic permissions are needed)
- Enable Push Notifications capability in Xcode (required for FCM)
- Verify `UIApplicationSupportsMultipleScenes` is `false` (already set — correct for this app)

### 5. CocoaPods / Dependency Installation
- Run `cd ios && pod install --repo-update` to install all Firebase and Flutter plugin iOS pods
- Resolve any pod version conflicts (Firebase 10.x+ pods should be compatible with the Dart SDK versions in pubspec.yaml)

### 6. Build & Test on iOS Simulator
- Run `flutter run -d ios` to verify the app builds and runs on an iOS simulator
- Test the full student flow: role selection → login → fee gate → OTP → test → result
- Test the admin flow: login → dashboard → results → push notifications
- Verify FCM token registration works on iOS (requires real device for full FCM test)

### 7. Build & Test on Real iOS Device
- Connect an iPhone via USB
- In Xcode, set the team for signing and run on device
- Test FCM push notifications (APNs requires a real device)
- Verify all Firebase services work: Firestore reads/writes, Cloud Functions calls, Auth

### 8. Create Archive & Upload to App Store Connect
- In Xcode: Product > Archive (select Any iOS Device)
- Validate the archive (Xcode will check signing, icons, etc.)
- Distribute App > App Store Connect > Upload
- Alternatively via CLI: `flutter build ipa --release` then upload with `xcrun altool` or Transporter app

### 9. App Store Submission
- In App Store Connect, fill in: app name (NSAT), subtitle, description, keywords, support URL, privacy policy URL
- Upload screenshots (iPhone 6.7", 6.5", 5.5" sizes required)
- Set pricing (free), availability, age rating
- Submit for Apple Review
- Address any rejection feedback from Apple review team

### 10. Post-Approval Release
- Set release type: manual release or automatic release upon approval
- Coordinate with the 14 June 2026 entrance test date
- Monitor crash reports via App Store Connect / Firebase Crashlytics (add if not already in project)
